Create tests/test_eval_integration.py implementing D-05's strategy:

```python
"""Integration test for the full evaluation pipeline.

Uses a synthetic checkpoint + synthetic ImageFolder (D-05):
1. Create synthetic ImageFolder (3 classes, 30 images, 32x32 PNG)
2. Initialize SimCLRv1Module with resnet18 backbone (random weights)
3. Save to .ckpt via trainer.save_checkpoint()
4. Run full eval pipeline: k-NN + linear_probe + t-SNE + UMAP + finetune + CAM
5. Assert all output files exist and no exceptions raised
"""
```

Key fixtures and tests:

1. `@pytest.fixture` `synthetic_data(tmp_path)`:
   - Create `tmp_path / "train"` and `tmp_path / "val"` directories.
   - For train: 3 classes, 10 images each (30 total), 32x32 RGB PNGs.
   - For val: 3 classes, 5 images each (15 total), 32x32 RGB PNGs.
   - Return tmp_path.

2. `@pytest.fixture` `synthetic_checkpoint(tmp_path, synthetic_data)`:
   - `import methods` to register all methods in dispatcher.
   - Create a TrainConfig dict: `{"method": "simclr_v1", "backbone": "resnet18", "max_epochs": 1, "batch_size": 4, "data_dir": str(synthetic_data), "n_views": 2}`.
   - `cfg = TrainConfig.model_validate(config_dict)`.
   - `from core.dispatcher import method_dispatcher`, `model = method_dispatcher(cfg)`.
   - Save checkpoint: use `torch.save({"state_dict": model.state_dict(), "hyper_parameters": {"cfg": cfg.model_dump()}}, ckpt_path)` or use Lightning's native save.
   - Actually, the cleanest way: `trainer = L.Trainer(max_epochs=0, enable_checkpointing=False, logger=False)`, then `trainer.strategy.connect(model)`, then save. Or simpler: just call `torch.save(...)` directly with the right keys.
   - Simplest approach that works with `load_from_checkpoint`: train for 0 or 1 step. Use `L.Trainer(max_steps=1, max_epochs=1, enable_checkpointing=True, default_root_dir=str(tmp_path), logger=False, enable_progress_bar=False)`. Set up data module, fit, get checkpoint from `tmp_path/lightning_logs/...` or save manually.
   - Alternative: `trainer.save_checkpoint(ckpt_path)` after connecting model. But this requires Trainer.fit or at least Trainer.test.
   - Most reliable: build model, save state_dict manually in Lightning checkpoint format:
     ```python
     ckpt_path = tmp_path / "test_checkpoint.ckpt"
     torch.save({
         "state_dict": model.state_dict(),
         "hyper_parameters": {"cfg": cfg.model_dump()},
     }, ckpt_path)
     ```
   - Return `(ckpt_path, cfg, synthetic_data)`.

3. `test_eval_config_schema_exists()`:
   - Verify FOUND-08: `from core.config import EvalConfig, LinearProbeConfig, KNNConfig, TSNEConfig, UMAPConfig, FinetuneConfig, CAMConfig`.
   - Assert all 6 sub-configs are importable and EvalConfig has the 6 optional fields.
   - `cfg = EvalConfig(knn=KNNConfig(), linear_probe=LinearProbeConfig())` -- constructs without error.

4. `test_knn_on_synthetic(synthetic_checkpoint)`:
   - Load model from checkpoint, extract features from train and val splits using the knn_callback's internal feature extraction or the standalone `knn_predict` function.
   - Call `knn_predict(train_feats, train_labels, val_feats, val_labels, k=5, temperature=0.07, num_classes=3)`.
   - Assert `acc >= 0.0` (D-05: relaxed threshold).

5. `test_linear_probe_on_synthetic(synthetic_checkpoint)`:
   - Load model, extract features using `extract_and_cache`, create LinearProbeModule, train for 2 epochs on cached features.
   - Assert training completes without error.
   - **Verify checkpoint-keyed cache filenames (D-04):** After calling `extract_and_cache(backbone, dataloader, cache_dir, "train", device, str(ckpt_path))`, derive `ckpt_stem = Path(ckpt_path).stem` and assert that cache files with the checkpoint stem prefix exist in cache_dir:
     ```python
     ckpt_stem = Path(ckpt_path).stem
     assert (cache_dir / f"{ckpt_stem}_features_train.pt").exists()
     assert (cache_dir / f"{ckpt_stem}_labels_train.pt").exists()
     ```

6. `test_tsne_on_synthetic(synthetic_checkpoint)`:
   - Load model, extract features (small subset), call `run_tsne(features, labels, [10], output_dir)`.
   - Use only 1 perplexity value (not 3) to minimize runtime.
   - Assert 1 PNG file exists with "perp" in filename.
   - Assert file size > 0.

7. `test_umap_on_synthetic(synthetic_checkpoint)`:
   - Load model, extract features, call `run_umap(features, labels, "cosine", output_dir)`.
   - Assert PNG file exists and size > 0.
   - Assert reducer is returned.

8. `test_finetune_on_synthetic(synthetic_checkpoint)`:
   - Load model from checkpoint, get backbone and feat_dim.
   - Create `FinetuneModule(backbone, feat_dim, num_classes=3, cfg)` where `cfg` has `eval.finetune` populated with minimal settings.
   - Create a small DataLoader from the synthetic ImageFolder (train split).
   - Train for 1 epoch using `L.Trainer(max_epochs=1, max_steps=2, logger=False, enable_progress_bar=False, enable_checkpointing=False)`.
   - Assert training completes without error (no accuracy threshold -- random weights).

9. `test_cam_on_synthetic(synthetic_checkpoint)`:
   - Load model, load 4 reference images from synthetic data as numpy arrays.
   - Call `run_cam(model, "resnet18", images, CAMConfig(n_images=4), output_dir)`.
   - Assert overlay PNG files exist.

10. `test_full_pipeline(synthetic_checkpoint)`:
   - Run all 6 above in sequence on the same checkpoint: kNN, linear probe, t-SNE, UMAP, finetune, CAM.
   - For linear probe step, verify checkpoint-keyed cache files exist (same assertion as test_linear_probe_on_synthetic).
   - Final assertion: all output directories contain expected files.

Mark integration tests with `@pytest.mark.slow` so they can be filtered. Use minimal parameters everywhere (k=5, 1 epoch for probe, 1 perplexity for t-SNE, 1 epoch / 2 steps for finetune, few samples for UMAP) to keep runtime under 120s. Note: 120s timeout exceeds the 60-second feedback latency guideline but is necessary for a 6-component synthetic integration test; individual component unit tests (Plans 01-04) still meet the 60s target.
