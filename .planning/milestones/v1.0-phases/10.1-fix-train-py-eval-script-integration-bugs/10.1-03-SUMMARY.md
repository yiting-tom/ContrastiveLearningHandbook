# Phase 10.1 Plan 03: Fix B1+B2+B5 in eval/linear_probe.py and eval/finetune.py Summary

Fixed three integration bugs (B1: ModuleNotFoundError, B2: SSLDataModule mis-construction, B5: silent NoneType crash on missing val/) in the two eval scripts by adding sys.path bootstrap, switching to explicit-kwarg SSLDataModule construction matching train.py, and inserting a FileNotFoundError guard before the val feature extraction.

## Changes Made

### eval/linear_probe.py

**B1 fix (sys.path bootstrap):** Added immediately after `from __future__ import annotations`:
```python
import sys
from pathlib import Path
sys.path.insert(0, str(Path(__file__).resolve().parent.parent))
```
Allows `python eval/linear_probe.py ...` from repo root without editable install.

**B2 fix (SSLDataModule construction):** Replaced `SSLDataModule(cfg)` with explicit-kwarg call mirroring train.py:52-57:
```python
dm = SSLDataModule(
    data_dir=cfg.data_dir,
    n_views=cfg.n_views,
    batch_size=cfg.batch_size,
    num_workers=cfg.num_workers,
)
```
The old call passed the entire TrainConfig as `data_dir`, silently corrupting DataModule state.

**B5 fix (val-loader guard):** Inserted before the val `extract_and_cache(...)` call:
```python
val_loader = dm.val_dataloader()
if val_loader is None:
    raise FileNotFoundError(
        f"Linear probe requires a 'val/' subdirectory under data_dir='{cfg.data_dir}'. ..."
    )
```
Updated the following `extract_and_cache` call to use the local `val_loader` variable.

### eval/finetune.py

**B1 fix (sys.path bootstrap):** Same pattern as linear_probe.py, added after `from __future__ import annotations`.

**B2 fix (SSLDataModule construction):** Same explicit-kwarg pattern replacing `SSLDataModule(cfg)`.

**Val-dataset guard preserved:** The existing guard at lines 213-217 (`val_ds = dm.val_dataset; if val_ds is None: raise RuntimeError(...)`) was intentionally left unchanged.
