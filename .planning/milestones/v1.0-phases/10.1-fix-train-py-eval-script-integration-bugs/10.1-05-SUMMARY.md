## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed multi-view batch extraction in eval/finetune.py training_step and validation_step**
- **Found during:** Task 2, running `test_finetune_accepts_train_checkpoint`
- **Issue:** `FinetuneModule.training_step` used `imgs[:, 0]` to extract first view from multi-view batches, assuming shape `[B, n_views, C, H, W]`. SSLDataModule's `ssl_collate_fn` actually produces `[n_views, B, C, H, W]`. The incorrect indexing gave shape `[n_views, C, H, W]` (2 images) vs labels shape `[B]` (4), causing `ValueError: Expected input batch_size (2) to match target batch_size (4)`.
- **Fix:** Changed `imgs[:, 0]` to `imgs[0]` in both `training_step` and `validation_step`. Also updated the comment to accurately describe the `[n_views, B, C, H, W]` shape.
- **Files modified:** `eval/finetune.py` (lines 93-94, 104-105)
- **Commit:** `7e09266`

**2. [Rule 2 - Missing functionality] vis tests need separate data_dir pointing to train/ subdir**
- **Found during:** Task 2, running `test_tsne_accepts_train_checkpoint` 
- **Issue:** `tsne_vis.py` and `umap_vis.py` call `ImageFolder(root=cfg.data_dir)` directly, expecting class subdirectories as immediate children. The smoke.yaml sets `data_dir=str(tmp_path)` which contains `train/`, `val/`, `train_logs/` — not class dirs. `ImageFolder` treated `train_logs/` as a class with no images.
- **Fix:** For SC-2b and SC-2c, created `vis_smoke.yaml` with `data_dir=str(tmp_imagefolder_with_val / "train")` pointing to the flat ImageFolder root.
- **Files modified:** `tests/test_e2e_pipeline.py` (test_tsne_accepts_train_checkpoint, test_umap_accepts_train_checkpoint)
- **Commit:** `7e09266` (included in Task 2 commit)
