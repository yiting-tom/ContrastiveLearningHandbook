Per D-08: Create `methods/swav/losses.py` with:

1. `sinkhorn_knopp(scores, n_iters=3, epsilon=0.05)`:
   - Decorated with `@torch.no_grad()`
   - Docstring: Args (scores [B, K], n_iters, epsilon), Returns Q [B, K], notes on doubly-stochastic property
   - Implementation (from official facebookresearch/swav):
     ```python
     Q = torch.exp(scores / epsilon).t()  # [K, B]
     B = Q.shape[1]
     K = Q.shape[0]
     Q /= Q.sum()
     for _ in range(n_iters):
         Q /= Q.sum(dim=1, keepdim=True)
         Q /= K
         Q /= Q.sum(dim=0, keepdim=True)
         Q /= B
     Q *= B
     return Q.t()  # [B, K]
     ```

2. `swav_loss(z_list, prototype_layer, temperature, n_large_crops, sinkhorn_fn)`:
   - Docstring: Args, Returns scalar loss
   - Implementation:
     ```python
     n_crops = len(z_list)
     scores = [prototype_layer(F.normalize(z, dim=1)) for z in z_list]
     loss = torch.tensor(0.0, device=z_list[0].device)
     for i in range(n_large_crops):
         q = sinkhorn_fn(scores[i].detach())
         for v in range(n_crops):
             if v == i:
                 continue
             p = F.log_softmax(scores[v] / temperature, dim=1)
             loss -= torch.mean(torch.sum(q * p, dim=1))
     loss /= n_large_crops * (n_crops - 1)
     return loss
     ```
   - CRITICAL: `.detach()` on sinkhorn output — codes are targets, not predictions
   - CRITICAL: Codes computed only from large crops (indices 0..n_large_crops-1)
   - All crops (including small) predict each large crop's codes
