2. Create eval/knn_callback.py with:
   - Imports: `from __future__ import annotations`, `import numpy as np`, `import torch`, `import torch.nn.functional as F`, `import lightning as L`, `from core.config import KNNConfig`
   - `knn_predict(train_features, train_labels, test_features, k, temperature, num_classes)` standalone function:
     - All inputs are torch tensors. train_features and test_features are [N, D] and [M, D] L2-normalized.
     - If `train_features.shape[0] > 100_000`: use FAISS path -- `import faiss`, create `faiss.IndexFlatIP(dim)`, `index.add(np.ascontiguousarray(train_features.numpy().astype(np.float32)))`, `similarities, indices = index.search(np.ascontiguousarray(test_features.numpy().astype(np.float32)), k)`, convert back to torch tensors.
     - Else: brute-force `similarities = test_features @ train_features.T`, then `similarities, indices = similarities.topk(k, dim=1)`.
     - Temperature-scaled weighted voting: `weights = (similarities / temperature).exp()`, `neighbor_labels = train_labels[indices]`, `votes = torch.zeros(M, num_classes)`, `votes.scatter_add_(1, neighbor_labels.long(), weights)`, `predicted = votes.argmax(dim=1)`.
     - Return accuracy as float: `(predicted == test_labels).sum().item() / M`. Note: test_labels must be a separate parameter.
     - Corrected signature: `knn_predict(train_features, train_labels, test_features, test_labels, k, temperature, num_classes) -> float`
   - `class KNNCallback(L.Callback)`:
     - `__init__(self, knn_config: KNNConfig)`: store `self.k = knn_config.k`, `self.temperature = knn_config.temperature`, `self.every_n_epochs = knn_config.every_n_epochs`.
     - `_should_run(self, trainer) -> bool`: If `self.every_n_epochs == 0`, return `trainer.current_epoch == trainer.max_epochs - 1`. Else return `(trainer.current_epoch + 1) % self.every_n_epochs == 0`.
     - `_extract_features(self, pl_module, dataloader) -> tuple[torch.Tensor, torch.Tensor]`: iterate dataloader with `torch.no_grad()`, call `pl_module.backbone(imgs.to(pl_module.device))`, collect features and labels, cat, L2-normalize features via `F.normalize(features, dim=1)`, return `(features.cpu(), labels.cpu())`.
     - `on_validation_epoch_end(self, trainer, pl_module)`: guard `if not self._should_run(trainer): return`. Get `val_loader = trainer.datamodule.val_dataloader()`. Guard `if val_loader is None: return`. Extract train features from `trainer.datamodule.train_dataloader()` (use the underlying dataset's non-augmented version -- actually, for k-NN we need the training set features too. Extract from both train and val loaders). Extract `(train_feats, train_labels)` from train_dataloader and `(val_feats, val_labels)` from val_loader. Infer `num_classes = int(train_labels.max().item()) + 1`. Call `knn_predict(...)`. Log via `pl_module.log("eval/knn_acc", acc, prog_bar=True)`.

   NOTE on train features: The train_dataloader returns augmented SSL batches (multiple views). For k-NN, we need single-view features. The simplest approach: build a temporary non-augmented dataloader from `trainer.datamodule.train_dataset` (the underlying ImageFolder) with a simple center-crop + normalize transform. However, to keep things simple and avoid modifying SSLDataModule, use the val_dataloader for BOTH train and val features in a "val-only k-NN" mode (common in tutorials). Actually, the standard protocol uses train features as the bank and val features as queries. The train_dataloader produces multi-view batches `[n_views, B, C, H, W]`. We can take just the first view `batch[0][0]` (first element of the views tuple). Let's handle this by: in `_extract_features`, if the batch is a tuple/list of tensors (SSL multi-view), take `imgs = batch[0]` for the images (first view) and `labels = batch[1]` for labels. If it's a standard `(imgs, labels)` tuple, use directly.

   Revised approach for train features: The `SSLDataModule.train_dataloader()` returns batches where:
   - For standard SSL: `(views, labels)` where views is `[n_views, B, C, H, W]` tensor
   - For indexed: `(views, labels, indices)` tuple
   
   In _extract_features, detect the format: if the first element is a tensor with ndim > 4, it's multi-view -- take `imgs = batch[0][0]` (first view). Otherwise `imgs = batch[0]`. Labels are always `batch[1]` (or `batch[-2]` for indexed). Simplest: always `imgs = batch[0][0] if batch[0].ndim == 5 else batch[0]`, `labels = batch[1]`.
