# Phase 10.1 Plan 04: Fix B1 (sys.path bootstrap) + B4 (perplexity clamp) in eval visualization scripts

**One-liner:** Applied sys.path.insert(0, repo_root) bootstrap to tsne_vis.py, umap_vis.py, and cam_vis.py, and added safe_perplexity = min(perplexity, n-1) clamp in tsne_vis.py:run_tsne to prevent sklearn >= 1.2 ValueError on small datasets.

## Summary

Fixed two integration bugs across three eval visualization scripts:

- **Bug B1** (`ModuleNotFoundError: No module named 'core'`): Added the identical three-line sys.path bootstrap after `from __future__ import annotations` in all three files. The bootstrap uses `sys.path.insert(0, str(Path(__file__).resolve().parent.parent))` so that `python eval/<script>.py` from the repo root can resolve `core` and `methods` packages without a `pip install -e .` editable install.

- **Bug B4** (sklearn >= 1.2 `ValueError: perplexity must be less than n_samples`): Added `safe_perplexity = min(perplexity, n - 1)` clamp inside the perplexity loop in `tsne_vis.py:run_tsne`. Also added an `n < 4` guard that raises a helpful `ValueError` for pathologically small datasets. Output PNG filenames still use the original requested perplexity for backward compatibility; a warning is printed to stdout when clamping occurs.

The `eval/dinov2_demo.py` file was verified to have no `from core` or `from methods` imports at module level — it is confirmed exempt from B1 and was not modified.
