<objective>
Wire SwAV and DINO to use MultiCropDataset (2×224 + 6×96 crops) in train.py instead of the default 8 uniform-size crops passed via n_views.

Purpose: SwAV and DINO require crops at two different spatial resolutions. Using a single n_views=8 with uniform size sends same-resolution crops to a multi-scale method, silently producing wrong training. MultiCropDataset applies ContrastiveAugmentation at two sizes independently and ssl_collate_multi_crop returns a list of tensors (not a stacked tensor) to handle the mixed spatial dimensions.

Output: Modified train.py with a shared swav/dino MultiCropDataset construction block.
</objective>

    if cfg.method in {"swav", "dino"}:
        from torchvision.datasets import ImageFolder
        import os as _os
        _train_dir = _os.path.join(cfg.data_dir, "train")
        if not _os.path.isdir(_train_dir):

        if cfg.method == "swav" and cfg.swav is not None:

        _wrapped_dataset = MultiCropDataset(

            n_large_crops=_n_large,
            large_size=_large_sz,
            n_small_crops=_n_small,
            small_size=_small_sz,
        )
