# 對比學習演化史 — Q&A 應答稿

> 用途：報告後 Q&A 速查表。每題已對照本專案程式碼與原始論文查核過，可直接照唸。⚠️ 標記的是最可能被釘、需要先想清楚的題目。每節已把最常被問的排在前面。

---

## 基礎直覺

**Q：對比學習跟一般的監督式分類，最根本的差別在哪？是不是只是把標籤換成增強而已？**
**A：** 可以這樣起頭，但有個關鍵差別。監督式學習有一組固定的「類別」，貓是貓、狗是狗，任務邊界是人定好的。對比學習沒有類別這個概念，它做的是「instance 級」的比較——每張圖先當成獨一無二的個體，模型只學「誰跟誰像、誰跟誰不像」這種相對關係。神奇的地方在於，你光靠這種相對關係硬學，語意結構居然會自己浮現出來——同類的圖最後自動聚在一起。所以它學的不是「分類」，而是一個好用的特徵空間，之後你想接分類、分割、什麼都行。

**Q：我還是不太懂，如果只是「同一張圖的兩個版本要拉近」，為什麼模型不會直接把全世界都壓成同一個點？坍塌到底是怎麼被擋下來的？**
**A：** 對，這就是整場報告的核心張力。如果你只有「拉近」這個力量，把所有圖都對應到同一點確實是最偷懶的滿分解，這就是坍塌。所以早期的解法是再加一個「推開」的力——不同的圖要互相推開，這樣大家就不可能擠在同一點。後來大家發現，推開不是唯一的辦法，你也可以用「打破對稱」來擋：像 BYOL、SimSiam 那一派，讓兩條分支不對稱（一邊多接一個 predictor），再對另一邊停掉梯度（stop-gradient），光靠這個就能防坍塌。記憶點就一句：防坍塌的本質，不是非得推開別人，而是不能讓兩邊塌成一模一樣。

⚠️ **Q：都說「沒有標籤也能學」，可是資料增強要決定怎麼裁切、怎麼變色，這些不也是人手設計的嗎？這樣到底算不算真的沒用到人類知識？**
**A：** 問得很好，這其實是最常見的誤會。我們拿掉的是「標籤」這根拐杖——沒有人去一張一張告訴模型這是貓那是狗。但資料增強確實是我們塞進去的一個人類先驗：我們在告訴模型「同一張圖換個角度、變個顏色，它還是同一個東西」。所以更精準的講法是：我們沒有給「答案」，但給了一條「不變的規則」。這也是為什麼大家說增強是對比學習的靈魂——你的增強 policy 選什麼，等於在定義模型該忽略哪些東西、該抓哪些東西。

⚠️ **Q：既然訓練到最後幾乎每個方法都能把群分出來，那研究這十幾種方法到底有什麼意義？是不是其實都差不多？**
**A：** 這個問題其實點到我跑完所有實驗最大的體會。「能不能分群」這件事，主要看你有沒有充分訓練——只要 epoch 夠，大部分方法最後都能分出來。但方法之間的差別，不在「能不能」，而在「用什麼代價、走哪條路去防坍塌」：要不要負樣本、要多大的 batch、收斂多快、夠不夠穩。打個比方，大家最後都能到山頂，但有人要扛一大袋負樣本上去，有人輕裝就上去了。這條演化史的價值，就是看人類怎麼一步步找到越來越省、越來越聰明的爬法。

⚠️ **Q：坍塌如果真的發生了，我在訓練的時候怎麼看得出來？是 loss 變高還是變低？**
**A：** 這裡要先分清楚兩種情況，因為它們的 loss 表現剛好相反。第一種是有負樣本的對比式方法（像 SimCLR、MoCo）：負樣本會擋住完全坍塌，所以真的坍塌時 loss 不會歸零，反而是卡在一個高的地板下不去（大約是 log(負樣本數) 那個量級）。第二種是 BYOL、SimSiam 這種沒有負樣本的方法：如果防坍塌機制失效，loss 反而會漂亮地往下掉、甚至直接歸零，看起來像訓練超好，其實是模型把所有東西壓成同一個點偷懶了——這才是最危險的陷阱。所以結論是：你不能只看訓練 loss。實務上要看別的訊號：把特徵畫成 UMAP，如果是一團毛球散不開；或是去算特徵每個維度的標準差，發現幾乎都塌到零、沒有變化（這正是 SimSiam 論文用來抓坍塌的指標），那就是坍塌。我們 demo 裡那張「訓練前一團 vs 訓練後分群」的對照圖，左邊那團毛球就是坍塌長的樣子。

**Q：InfoNCE 那個溫度 τ，你說它調鬆緊，可是調大調小到底差在哪？設錯會怎樣？**
**A：** 你可以把 τ 想成放大鏡的倍率。τ 設很小，等於把相似度差距放得很大，模型會死盯著那些「長得很像但其實是不同圖」的難負樣本猛推，學到的特徵很細、區分度高，但也對難負樣本特別敏感、訓練容易不穩。τ 設很大，差距被抹平，模型對誰都溫和，結果學出來的特徵就比較糊、分得不夠開。所以它是一個甜蜜點，要去調的。我們這個專案 SimCLR/InfoNCE 的預設值就是 0.5，而 MoCo、SupCon 那些方法常用更尖銳的 0.07，沒有一個萬用的數字。

**Q：你一直說增強很重要，那是不是增強越狠、變化越大，學出來的特徵就越好？有沒有上限？**
**A：** 不是越狠越好，這正好是 InfoMin 那篇在講的事。它的精準說法是「最小充分（minimal sufficient）」：重點不是增強的力道，而是兩個視角之間「共享的資訊量」要剛剛好——該共享的語意要全部留住（充分），不該共享的捷徑像顏色、紋理就盡量砍掉（最小）。如果增強太弱，兩個視角共享太多，模型會偷懶，靠顏色、靠背景這種捷徑就把它們配對起來，根本沒學到語意；但如果增強太狠，狠到把貓都裁成只剩一隻耳朵，兩個視角連語意都不共享了，模型反而學壞。所以視角之間的互資訊其實是一條 U 型曲線，有個最佳點。瓶頸有時候不在 loss、不在架構，而在你餵進去的「視角」本身。順帶一提，InfoMin 的 loss 本身就是 SimCLR 那套 NT-Xent，差別只在它用這個「最小充分」原則去挑視角。

**Q：負樣本就是「不同的圖」，那萬一我從資料裡隨機抓的負樣本，剛好也是一隻貓呢？這樣硬把兩隻貓推開，不會把模型教壞嗎？**
**A：** 你抓到了一個真實存在的問題，學界叫它「假負樣本」——你以為是負樣本，其實語意上是同類。在沒有標籤的設定下，我們確實沒辦法百分之百避開它。為什麼整體還 work？第一，負樣本量很大的時候，少數幾個抓錯的影響會被稀釋掉；第二，溫度 τ 跟損失設計也讓模型不會用全力去推那些其實很像的對象。但你說的沒錯，這是對比式方法天生的雜訊。也正因為這個煩惱，後來像 BYOL、SimSiam 那一派乾脆連負樣本都不要了，從根本繞開這個問題。

## Era 2 · 負樣本從哪來

**Q：memory bank 跟 MoCo 的 queue，聽起來都是「把特徵存起來當負樣本」，差別到底在哪？**
**A：** 差別在「新不新鮮」跟「一不一致」。memory bank（Wu et al. 2018 的 instance discrimination）是整個資料集每張圖各配一格，你這一輪用到它，才順手把它更新一次——所以某一格上次被寫進去，可能是「一整個 epoch 之前」的事了，而且是更舊的模型算的，等於拿過期的特徵在比。MoCo 的 queue 是一條 FIFO 排隊，只裝最近幾個 batch 的 key（MoCo 預設約 65536 個，大概幾百個 batch），舊的會被擠掉，所以永遠是「最近才算的」。更關鍵的是 MoCo 那邊用動量編碼器慢慢更新，queue 裡的特徵雖然不是當下最新，但因為編碼器變得很慢、很平滑，彼此一致、不會亂跳；memory bank 沒這個機制，所以它的負樣本是又舊又不一致的。

**Q：SimCLR 那派靠大 batch，所謂「大」到底要多大？為什麼一般實驗室玩不起？**
**A：** 重點是 SimCLR 的負樣本完全來自同一個 batch。一個 batch 有 N 張圖、每張各做 2 個增強視角，所以共 2N 個樣本，對某一個視角而言可用的負樣本是 2N−2 個（扣掉自己跟自己的正樣本配對）。所以 batch 一小，負樣本就跟著餓死；SimCLR 論文顯示 batch 越大越好，要衝高還得換 LARS 這種大 batch 專用的優化器，他們最好的設定用到 batch size 4096、還訓練很多 epoch。問題是這種大 batch 要一大片 TPU 或一堆高階 GPU 才塞得下，記憶體直接爆掉，所以小實驗室玩不起。這正是 MoCo 的賣點——它用 queue 把負樣本數量跟 batch size 脫鉤，batch 很小照樣有上萬個負樣本。

**Q：為什麼負樣本「過時」就不行？舊一點的負樣本拿來推開，有差嗎？**
**A：** 有差，而且差在「一致性」與「對比的力道」。對比學習是拿你現在這一版模型算出來的特徵，去跟負樣本比相似度。如果負樣本是好幾輪前、由差很多的舊模型算的，那它跟現在的特徵幾乎是兩個座標系，比出來的相似度沒有意義、訊號就鈍掉了。MoCo 的論文其實點出關鍵不只是「新鮮」而是「一致」：用動量編碼器讓 key 的表示緩慢且平滑地變化，queue 裡前後的 key 才彼此可比。所以重點是：負樣本要夠多，而且要由一個跟得上、但又夠平滑一致的編碼器產生，推開才有意義。

**Q：MoCo 那個動量編碼器到底在動量什麼？它跟主編碼器是同一個嗎？**
**A：** 它是主編碼器的「慢動作影分身」。主編碼器（query 那邊）照常用梯度更新、學得快；動量編碼器（key 那邊）不吃梯度，它只是每一步都把主編碼器的權重「抄一點點過來」，公式是 θ_k ← m·θ_k + (1−m)·θ_q，m 設 0.999 時就是 0.999 乘舊的 key 編碼器加 0.001 乘主編碼器，所以它跟得很慢、很平滑。為什麼要慢？因為 queue 裡的特徵是它陸陸續續算出來的，如果它變太快，前後算的 key 就會打架、不一致。MoCo 論文裡動量設 0.999 最好，設小（像 0.9）就明顯變差，甚至更小會訓練不穩，慢一點反而穩，這點蠻反直覺的。

⚠️ **Q：SwAV 說它不用「成對」負樣本，可是它去比那些 prototype，prototype 不也算一種負樣本嗎？**
**A：** 問得很好，這是最容易混淆的點。差別在「跟誰比」。傳統對比是圖跟圖兩兩硬比，你得湊一大堆別張圖當負樣本。SwAV（Caron et al. 2020）不直接比圖，它設一組可學習的 prototype（可以想成一組群心向量），每張圖算出自己對這些 prototype 的軟分配（code），然後做 swapped prediction：用一個視角的特徵去預測另一個視角的 code。它從頭到尾沒有去枚舉成對的負樣本，所以說它「免成對負樣本」。你可以說 prototype 隱含了對比的味道，但它不是一張張具體的負樣本圖，而是一組共享的、抽象的群心，這就是它跟 MoCo、SimCLR 路線上的根本不同。要注意 prototype 是端到端學出來的，不是另外跑 k-means。

⚠️ **Q：SwAV 不用負樣本來推開，那它靠什麼防坍塌？不會所有圖都被分到同一群嗎？**
**A：** 它防坍塌的關鍵在算 code 時的 Sinkhorn-Knopp 這一步。如果放任不管，模型最偷懶的解就是「把所有圖都丟進同一個群」，那就坍塌了。Sinkhorn-Knopp 演算法做的事情，是在求解 batch 內分配時加上「均勻攤平」的約束——數學上是讓那個分配矩陣逼近雙隨機（等價於要求每個 prototype 平均被分到的量差不多），不准某一群獨大。要補一句：它是用在產生目標 code（target，不回傳梯度）那一側，當成軟性的最佳化分配。這等於從另一個角度堵住坍塌：你想全擠到一群？不行，我規定每群都要有人。所以對比學習的「推開」在這裡被換成了「強迫均勻分散」，殊途同歸。

⚠️ **Q：MoCo 跟 SimCLR 後來到底誰贏了？現在大家還在用大 batch 嗎？**
**A：** 與其說誰贏，不如說兩邊的好點子被互相吸收了。MoCo v2 就很乾脆，直接把 SimCLR 驗證有效的幾樣東西——MLP 投影頭、更強的增強（含高斯模糊）、cosine 學習率——搬進自己的框架，用小資源就追上甚至超過大 batch 的 SimCLR。有趣的是到了 Transformer（ViT）時代，MoCo v3 反而把自己招牌的 queue/memory queue 拿掉了，改用 batch 內互當負樣本的對比 loss，因為那時用的 batch 已經夠大（論文用到 4096）。所以這場「負樣本從哪來」的對決，結論比較像：兩種哲學都成立，看你手上有多少記憶體，缺記憶體就用 queue，記憶體夠就直接堆 batch。

**Q：SwAV 的 multi-crop 是另外加的特效，還是它能不用負樣本的關鍵？**
**A：** 兩件事要分開。能不用成對負樣本，靠的是 prototype 加 swapped prediction、再加 Sinkhorn 防坍塌那套機制；multi-crop 是另外一個「加菜」的技巧（而且它不只屬於 SwAV，可以搬到別的方法上用）。它的做法是一張圖切兩塊大的（高解析，如 2×160 或 2×224），再切好幾塊小的（低解析，如 6×96），小塊算很便宜，等於用相對低的成本多塞了好幾個視角去互相預測，提升學習效率。訣竅是：只有大塊負責產生「答案」code（會過 Sinkhorn），小塊不拿來算 code，只能當「猜題者」去預測大塊的 code。所以 multi-crop 是表現上的助推器，不是防坍塌的本體，這兩個貢獻常被混在一起。

## Era 3 · 無負樣本

⚠️ **Q：predictor 跟 EMA 兩個都拿來防坍塌，到底哪一個才是真正關鍵？是不是缺一不可？**
**A：** 這題很漂亮，而且答案有點反直覺。BYOL 是 predictor 加 EMA target 兩個一起上；但後來 SimSiam（Chen & He, CVPR 2021）做了一個最小化實驗，直接把 EMA 整個砍掉、兩條分支共用同一組權重，只留 predictor 加 stop-gradient，居然也不坍塌。所以結論是：真正不可少的是「predictor 的不對稱 + 那一刀 stop-gradient」，EMA 比較像是讓訓練更穩、表現更好的加分項，而不是防坍塌的必要條件。可以記成一句：EMA 是錦上添花，stop-gradient 才是命根子。

⚠️ **Q：stop-gradient 講起來很玄，實作上到底是停在哪一邊？如果我手滑忘了停會怎樣？**
**A：** 實作上就是在被當「目標」的那一端做 detach，讓梯度只從 predictor 那條路回傳，不流回 target 分支。在 SimSiam 裡梯度停在 z（projector 的輸出，也就是 cosine loss 裡的 z2.detach()/z1.detach()）那側；在 BYOL 裡 target 分支本來就用 EMA、不接優化器，還額外 detach 一次，雙重保險。這一刀是 load-bearing 的——只要忘了那個 detach，SimSiam 的對稱結構就只剩「拉近」這一招，會直接塌成同一個點，negative-cosine loss 掉到 -1.0 那條死線（這寫在 SimSiam 的 docstring 警告裡）。我們專案有對 BYOL 寫單元測試（test_byol.py）直接驗證 target 分支梯度為 0、online 分支有梯度，就是專門盯這個機制有沒有被破壞。

**Q：BYOL 既然沒有負樣本，那如果我只是把同一張圖的兩個視角拉近，模型為什麼不會直接把所有圖都對應到同一個點、loss 歸零？**
**A：** 問得超到位，這正是大家當年最怕的「坍塌」。關鍵在於 BYOL 故意把兩條分支做成不對稱：online 那邊多接一個叫 predictor 的小網路，target 那邊沒有；而且 target 端切斷梯度（stop-gradient）、只靠 EMA 慢慢跟上 online 的權重。因為兩邊不對等，常數解在這個不對稱結構下並不是個吸引子（attractor），模型沒辦法靠「大家一起塌成同一個常數」去偷懶把 loss 壓到 0。所以它防坍塌靠的不是「推開別人」，而是「打破對稱 + stop-gradient」，這也是 Era 3 整段最核心的收束句。（補充：坍塌解的 loss 並不是嚴格的全域最小，但在這個動態下訓練不會被它吸引過去。）

**Q：你怎麼知道訓練到一半有沒有偷偷坍塌？有沒有一個指標可以盯著看？**
**A：** 有，而且很簡單：盯著 embedding 在 batch 維度上的標準差（z.std(dim=0).mean()）。如果所有圖都塌成同一個點，各維度在 batch 內就沒有變化，這個 std 會往 0 掉，這是放諸四海皆準的坍塌警報。要注意數值高低跟你監看的是「正規化前還是後的向量」有關：SimSiam 論文監看的是 L2-normalized 後的輸出，健康值大約落在 1/√d（d 是嵌入維度）那個量級；我們專案裡 BYOL/SimSiam 記的是 projector 原始輸出的 std，看的是「有沒有往 0 滑」這個趨勢，而不是去比對某個固定數字。Barlow Twins 則是看互相關矩陣的對角線平均（corr_diag_mean），往 0 掉就代表兩個視角不再相關、開始塌了（我們用 0.5 當警戒線）。所以坍塌不是只能事後才發現，訓練當下就有儀表板可以看。

**Q：Barlow Twins 完全不玩拉近推開，那它的 loss 到底長怎樣？為什麼逼一個矩陣變單位矩陣就能不坍塌？**
**A：** 它看的是兩個視角嵌入之間的「互相關矩陣 C」（把兩批投影輸出沿維度標準化後，C = Z1ᵀ Z2 / B），然後 loss 就兩項：對角線逼近 1、非對角線逼近 0。公式是 Σ_i (1 − C_ii)² + λ · Σ_{i≠j} C_ij²，λ 預設 5e-3（Zbontar et al., ICML 2021）。對角逼近 1 是要「同一張圖的兩個視角在每個維度上要一致」，這是不變性；非對角逼近 0 是要「不同維度之間去相關、不要重複講同一件事」，這叫去冗餘。妙就妙在：如果模型偷懶把所有維度都塌成一樣，那些維度彼此高度相關，非對角會爆高、loss 反而被狠狠懲罰，所以去冗餘這一項本身就在頂住坍塌——它沒有負樣本、沒有 EMA、沒有 predictor，功夫全下在 loss 上。（實務上投影頭維度要開大，我們用 8192。）

**Q：SwAV、BYOL、Barlow Twins、DINO 都說自己不用負樣本，那它們到底差在哪？不就都是同一招換包裝？**
**A：** 表面上都是「不用負樣本」，但它們打破對稱、頂住坍塌的手法完全不同，這也是收束表想講的事。SwAV 靠 prototypes 加 Sinkhorn-Knopp 把線上分配軟性地均勻攤平（swapped prediction：用 A 視角的特徵去預測 B 視角的分配），不准全擠到同一群——注意它是線上 Sinkhorn 軟分配，不是離線 k-means；BYOL 跟 SimSiam 靠 predictor 不對稱加 stop-gradient；Barlow Twins 靠去冗餘的 loss；DINO 靠 teacher 端的 centering 加 sharpening（EMA teacher 的自蒸餾）。機制各走各的路，但骨子裡回答的是同一個問題——沒有「推開別人」這根拐杖，我要怎麼不讓兩邊塌成一模一樣。所以不是換包裝，是同一個敵人、好幾種不同的武器。

⚠️ **Q：BYOL 跟 SimSiam 看起來只差一個 EMA，那實務上我該選哪一個？是不是 SimSiam 比較簡單就一定比較好？**
**A：** 可以這樣看：SimSiam 是 BYOL 拿掉 EMA target 的極簡版，它最大的價值是「實驗證明」——告訴你防坍塌的關鍵是 predictor + stop-gradient，不是 EMA。但簡單不代表全面更強。BYOL 的 EMA target 是一個比較穩、變化比較平滑的學習目標（動量通常還用 cosine 從 0.996 排程到 1.0），通常最終表現會更好一點、也更耐操。另外兩者超參數脾氣不太一樣，SimSiam 論文用 SGD（不是 LARS），base lr 還跟著 batch size 線性縮放（lr = 0.05 × batch_size / 256）。所以一句話：想讀懂原理、想要乾淨的最小例子就看 SimSiam；想追效果和穩定度，BYOL 那個 EMA 還是值得留著。

**Q：這些不用負樣本的方法，聽起來都好脆弱，是不是隨便調個超參數就坍塌？它們可靠嗎？**
**A：** 其實沒有想像中那麼玻璃心。只要該有的不對稱在——predictor、stop-gradient、EMA（BYOL）或去冗餘 loss（Barlow Twins）擺對位置——它們是能穩穩訓練到底的，我們在 H100 上自己跑兩百個 epoch，BYOL 跟 SimSiam 的特徵都漂亮地分成群，完全沒塌。真正脆弱的地方很集中：忘了那一刀 stop-gradient、predictor 接錯邊、或 Barlow Twins 把投影頭維度開太小（它要到 8192 那種高維才好用）。換句話說，不是整體脆弱，而是「打破對稱的那個機制」一旦被破壞就會立刻坍塌。把那一根顧好，它們其實相當可靠。

## Era 4 · Transformer 與基礎模型

⚠️ **Q：DINO 的 attention 自己畫出物體輪廓，沒人教它分割。它到底是怎麼無中生有學會「物體」的？這是 Transformer 才有的特異功能嗎？**
**A：** 這是 DINO 最迷人的「啊哈」時刻，但要講清楚：它沒有真的學會分割，它只是注意力剛好亮在物體上。原因是 multi-crop——它會餵一張圖的小裁切（local view），去對齊整張圖的全局視角（global view）。要讓一個 96×96 的小局部跟整隻狗的特徵對得起來，模型唯一的辦法，就是去抓那個「不管你怎麼裁都還在的主體」，而背景常常被裁掉、對不上，自然就被忽略。ViT 的 self-attention 又天生會把 token 之間的關係攤開來，所以這個「鎖定主體」的行為，就直接顯影在 [CLS] token 的 attention map 上。換成 CNN，類似的語意也學得到，只是沒有這麼一張漂亮的注意力圖可以直接看。（補一句：後來的研究還發現這張圖會被少數高範數的背景 token 弄髒，要靠 register token 來清掉，這是後話。）

⚠️ **Q：DINOv2 既然只是把方法放大，為什麼大家還是把它當基礎模型？它跟 CLIP 這種也很強的視覺模型差在哪？**
**A：** 差在「資料來源」和「監督訊號」。CLIP 是拿幾億組網路上的「圖片＋文字說明」去對齊，本質上還是靠人類寫的文字當弱監督，而且它的特徵偏向「能不能用語言描述」。DINOv2 完全不用任何文字、任何標籤，純靠圖片自己學（在精心整理過的 LVD-142M 資料集上），所以它抓的是更純粹的視覺結構。實務上的結果是：DINOv2 的特徵在密集型任務——分割、深度估計、找對應點——特別強，常常凍住 backbone、不微調就能直接接一個小頭打贏；CLIP 則在「跟語言有關」的零樣本分類上更猛。一句話：CLIP 懂「圖配字」，DINOv2 懂「圖本身的幾何與結構」。

⚠️ **Q：DINO 之後是 DINOv2，那有沒有 DINOv3？這條線接下來往哪走？**
**A：** 有的，而且要更新一下血緣：DINO → DINOv2 →（中間有個重要插曲：register token，出自 2023/09 上傳、拿下 ICLR 2024 Outstanding Paper 的《Vision Transformers Need Registers》，用幾個額外 token 去吸收注意力裡的高範數雜訊、讓 attention map 更乾淨）→ DINOv3。DINOv3 是 Meta 在 2025/08 釋出的（arXiv 2508.10104），把規模再往上推到 17 億張影像、70 億參數的 teacher，並提出 Gram Anchoring 來解決「長時間訓練時 dense 特徵會慢慢退化」的老問題，所以在分割、深度、影片追蹤這些密集任務上又上一層樓。所以別再說沒有 DINOv3 了。至於趨勢，就是我們整場的結論：用越來越少的人為假設、餵越來越大的資料，讓一個通用視覺基礎模型自己長出來，然後凍住它、到處接各種下游任務——這條無監督的線，正在跟語言模型那邊的 foundation model 浪潮匯流。

**Q：前面那些 ResNet 的方法，直接把 backbone 換成 ViT 不就好了？為什麼非得專門研究 MoCo v3、還搞出一堆穩定技巧？**
**A：** 因為換引擎不是隨插即用，ViT 很「嬌」。CNN 自帶兩個內建的拐杖——卷積的局部性、還有 pooling 的平移容忍，這讓它天生比較穩。ViT 把這些先天歸納偏置（inductive bias）幾乎全拿掉了，等於更自由、但也更容易在訓練早期翻車。所以 MoCo v3 的這篇實證研究（He 團隊，2021，arXiv 2104.02057）主要靠兩招去馴服它：第一是把 patch embedding 凍結成隨機投影，直接壓掉 loss 爆衝；第二是把優化器從大批次常用的 LARS 換成帶 weight decay 的 AdamW。論文裡也試過梯度裁剪等手段，但真正關鍵的就是那兩招。這其實又是我們的主線——拿掉 CNN 這根架構拐杖之後，你得補上一整套新的穩定配方，它才站得住。

**Q：為什麼偏偏是 patch embedding 那一層在搞鬼？凍住它不會讓模型少學到東西嗎？**
**A：** 問得好。patch embedding 就是把圖片切成小方塊、再用一個小小的卷積投影成向量的那一層，它是整個 ViT 的入口。MoCo v3 團隊發現，訓練時這層的梯度會突然爆衝（一個 spike），loss 緊接著就出現一個 dip、整條曲線跟著抖。而且這個爆衝最早就發生在第一層，過幾十個 iteration 才傳到後面的層。直覺上的解釋是：這層離輸入最近、最好動手腳，優化器會偷懶，直接在這裡做出一堆「廉價但好區分」的 patch 表徵，而不是逼後面的 Transformer 去學真正的語意——注意，MoCo v3 是有負樣本的對比學習，所以這不是真的坍塌，而是一種訓練不穩、走捷徑的傾向。把它凍結成隨機初始化，等於把這條偷懶的路堵死，逼 Transformer 自己扛起表徵的重活。而且實測幾乎不掉準度（反而還小漲），因為 patch projection 的輸出維度通常是 complete 甚至 over-complete 的，一個隨機線性投影本來就足以保住原始資訊。

**Q：DINO 的 centering 和 sharpening 聽起來方向相反，一個拉平、一個變尖，這樣不會互相抵消嗎？它們到底怎麼合作防坍塌？**
**A：** 對，它們刻意是一對相反的力，妙就妙在這裡。坍塌有兩種長相：一種是所有圖都被分到同一個 prototype（擠成一點），另一種是輸出變成平平的均勻分布、什麼都沒講。centering 是減掉 teacher 輸出的滑動平均（EMA），專門打壓「某一維/某一類獨大」，把它往均勻方向推、避免擠到同一點；sharpening 是用很低的溫度把分布變尖，逼模型「做決定」、避免變成一坨均勻的爛泥。一個防擠成一點、一個防攤成一片，兩股力卡在中間，模型就被夾在一個剛剛好、不坍塌的甜蜜點上。這也呼應我們的主線：DINO 不是靠推開負樣本（它根本沒有負樣本），而是靠這組設計打破對稱、撐住不塌。

**Q：DINOv2 加的 iBOT 那個「patch 級遮罩」，聽起來跟 MAE 把圖片挖洞重建是不是同一招？它跟 BERT、跟 MAE 的關係是什麼？**
**A：** 概念上是同一個家族，但目標不一樣。iBOT 確實像影像版的 BERT：把一部分 patch 遮起來，叫模型去預測那些被遮的位置。但關鍵差別在「預測什麼」。MAE 是去重建被遮掉的原始像素，目標是把畫面補回來；iBOT 是去預測被遮 patch 在 teacher 那邊的「特徵分布」（一樣是 EMA teacher 的自蒸餾），也就是把 DINO 的自蒸餾從整張圖的 [CLS]，延伸到每一個 patch token。所以 DINOv2 等於同時做兩件事：DINO loss 管影像級的整體語意，iBOT loss 管 patch 級的局部細節。它跟 MAE 是兩條平行路線——一條重建像素、一條對齊特徵，而 DINOv2 走的是後者。

**Q：DINO 的 teacher 就是 student 的影分身，自己教自己，那它一開始兩邊都是亂的，怎麼可能教出有用的東西？不會越教越爛嗎？**
**A：** 這正是 BYOL、DINO 這類方法最反直覺、也最像魔術的地方，叫做 bootstrap、自我拉拔。關鍵是 teacher 不是直接複製 student，而是 student 的「慢動作平均」——用 EMA 一點一點跟上。所以 teacher 永遠比 student 稍微穩、稍微好一咪咪，student 去追這個稍微好一點的目標，追上之後 teacher 又被往前推一點，兩邊就這樣螺旋著一起往上爬。但光有 bootstrap 還不夠，防坍塌另有功臣：DINO 靠的是 centering + sharpening，BYOL 則是靠 predictor 加 stop-gradient 製造的不對稱（順帶一提，SimSiam 後來證明連 EMA 都拿掉、只留 stop-gradient + predictor 也不會塌）。有了這些在旁邊看著，整個過程就能穩穩地從噪音裡長出結構，而不是越教越爛。

## 實務與本專案

⚠️ **Q：你 UMAP 上分群分得很漂亮，這能代表這些方法「學得好」嗎？還是只是 UMAP 自己把圖畫好看的？**
**A：** 這是最該小心的一點，我自己也踩過。UMAP 是非線性降維，它本來就傾向把點擠成一團一團，所以「看起來有分群」不能直接當成效能證據——換個參數、換個 random seed，形狀都會變。我這邊有固定 random_state=42、用 cosine 距離，讓圖至少可重現，但它頂多是個直覺輔助。真正要比好壞，還是得看 linear probe 或 kNN 的數字。所以請把 UMAP 當成「看防坍塌有沒有成功」的眼睛，而不是排名表。

⚠️ **Q：你 Live Demo 那張「訓練前 vs 訓練後」的對照圖，右邊真的是你自己訓練出來的 SSL 特徵嗎？**
**A：** 要老實講：那一張核心對照圖的「訓練後」，其實是用 ImageNet 預訓練的 ResNet-18 backbone 當代理，不是我自己從零訓的 SSL checkpoint，純粹是為了現場 demo 穩定、一定跑得出漂亮結果。但前面每個方法頁的那一整排 UMAP 演進帶——從 Instance Discrimination、Invariant Spread、SimCLR、InfoMin、BYOL、SimSiam，到 MoCo v1/v2/v3、SwAV、Barlow Twins、DINO，總共十三條自監督方法——那才是我在 H100 上真的從零訓、每個 epoch（0 到 200）拍一張快照接成的動畫。換句話說，要看「我親手跑的」，請看演進帶；那張 before/after 是教學示意，程式裡（tools/make_demo_umap.py）也有註明它是 proxy，換上自己的 checkpoint 用 eval/umap_vis.py 就能重現。

⚠️ **Q：你的 CIFAR-10 數字，跟論文 ImageNet 上報的那些漂亮結果能比嗎？可信度有多高？**
**A：** 不能直接比，要先講清楚定位。原始論文幾乎都在 ImageNet 上、用 ResNet-50、大 batch、跑好幾百甚至上千 epoch；我是 CIFAR-10、ResNet-18、batch 256、影像壓到 96px，目標是「讓大家看懂每個方法怎麼運作、怎麼防坍塌」，不是刷榜。所以我的結果可信的部分是「定性」——例如 BYOL、SimSiam 沒有負樣本也真的沒坍塌，這個現象在我的圖上看得很清楚；但「絕對準確率誰高誰低」這種排名，因為規模差太多，不能拿來當論文等級的結論。這是教學重現，不是 benchmark。

**Q：你說「充分訓練後幾乎每個方法都能分群」，那這些方法到底差在哪？我實務上該選哪一個？**
**A：** 對，這是我跑完所有實驗最大的體會：能不能分群，主要看你有沒有訓夠；方法的差別不在「能不能」，而在「用什麼代價、走哪條路去防坍塌」。實務上我會這樣選：算力有限、單卡小 batch，就走 MoCo 系或乾脆 BYOL、SimSiam，因為它們不靠超大 batch 也能 work（BYOL 仍要一個 EMA target 網路，SimSiam 則證明連 EMA 都可以拿掉、只靠 stop-gradient + predictor）；有大卡能開大 batch，SimCLR 最簡單直接，它就是吃 in-batch 負樣本；要 ViT 或想要能畫出物體輪廓的 attention，就 DINO；只是想要一組現成好特徵接下游，那根本別自己訓，直接拿 DINOv2 的官方權重。一句話：先看你有多少卡，再看你要不要負樣本。

**Q：你怎麼評估「學得好不好」？linear probe 跟 kNN 差在哪，為什麼要兩個都做？**
**A：** 兩個都是在「不動 backbone」的前提下，看凍結的特徵到底好不好用。linear probe 是把 backbone 凍住、只在上面訓一個線性分類頭（我這邊 weight decay 設 0），看純線性可不可分。kNN 更乾脆，連分類頭都不用訓，直接拿特徵算最近鄰投票，我用的是 DINO/MoCo v3 那套溫度加權的 kNN（k=200、temperature=0.07）。差別在：linear probe 要再訓一輪、比較貼近正式用法；kNN 零訓練、快，適合訓練中每幾個 epoch 順手量一次看趨勢。兩個一起看，可以分辨「特徵本身好」還是「只是剛好線性頭硬湊出來的」。

**Q：為什麼選 UMAP 不選 t-SNE？兩個不是都在做降維嗎？**
**A：** 專案裡其實兩個都有，t-SNE 也留著。選 UMAP 當主力有兩個原因：一是它比較快，而且一般認為在我們上千、上萬個點的規模下比較保得住「全局結構」，不像 t-SNE 那麼容易只顧局部、把大尺度的關係拆散（這點學界仍有爭論，所以別當成鐵律）。二是 UMAP 那個 reducer 物件可以用 .transform() 去 map 新樣本，做動畫、做演進帶很方便。但記住一句：不管 UMAP 還是 t-SNE，軸都沒有物理意義，點跟點的絕對距離也不能太認真——它就是個給人看的投影，不是評分。

**Q：你是怎麼在 H100 上訓的？CIFAR-10 配 ResNet-18 跑一輪要多久，很燒卡嗎？**
**A：** 我在學校兩張 H100 上跑，CIFAR-10 配 ResNet-18，從零訓 200 個 epoch、batch 256。關鍵省時間的招是把輸入解析度壓到 96px——這樣一張 H100 大概 30 秒就跑完一個 epoch；如果用標準 224px，會慢差不多十倍，而且瓶頸常常卡在 CPU 做資料增強，不一定是 GPU。我同時開多條訓練線、一張卡塞兩條，小方法一輪一輪很快就堆完了。所以聽起來很猛，其實 CIFAR-10 這種小資料配小模型，負擔沒大家想的那麼可怕。

**Q：別人拿你的 repo 重跑，能得到一模一樣的圖嗎？可重現性做到什麼程度？**
**A：** 結構上可重現性做得滿硬的：每個實驗就是一個 YAML 設定檔，所有方法共用同一支訓練腳本、同一套資料模組跟評估工具，UMAP 也固定了 random seed 42。所以「拿同一個 checkpoint、重跑評估跟畫圖」，基本上會得到一樣的結果。但要誠實說，「從零再訓一次」很難分毫不差——GPU 上多執行緒、cuDNN 的非確定性、資料增強的隨機性都會讓最後幾個小數點飄。所以我敢保證的是「同一條配方、同樣的趨勢與結論可重現」，而不是「每個像素都 bit-for-bit 一樣」。

**Q：DINOv2 你為什麼不自己訓？是訓不動，還是另有原因？**
**A：** 純粹是訓不動，也不該訓。DINOv2 要從零複現，得餵它一個一億四千多萬張、精心篩過的資料集 LVD-142M，還要好幾百個 GPU-day 的算力，這不是一場課堂 demo 扛得起的。所以這個專案對 DINOv2 只做「拿官方權重來抽特徵、微調」的 demo，不做從零訓練。順帶一提 DINOv2 的方法本身其實是混血：影像級用 DINO 的自蒸餾，patch 級再加上 iBOT 的 masked 預測，再用 LVD-142M 規模化堆起來。這也呼應結尾那句話：到了基礎模型這一步，精神已經從「自己訓」變成「大力出奇蹟之後，大家共用同一組權重」。

**Q：既然有 DINOv2 這種現成又強的官方權重，那自己訓這些舊方法還有意義嗎？**
**A：** 如果你的目標只是「拿一組好特徵去接下游任務」，老實說別自己訓，直接用 DINOv2 官方權重又快又強。但這個專案的價值不在拚效能，而在「看懂演化史」：你親手把 Instance Discrimination、SimCLR、BYOL 一個一個訓起來，才會真的體會到那條主線——這幾年大家在一根一根拿掉拐杖、用一堆不同辦法回答同一個問題「沒有標籤怎麼不讓表徵坍塌」（SimCLR 用 in-batch 負樣本、MoCo 用動量編碼器加 queue、BYOL/SimSiam 靠 stop-gradient 加 predictor、SwAV 用 Sinkhorn 軟分配加 swapped prediction、DINO 用 centering+sharpening、Barlow Twins 把互相關矩陣逼近單位矩陣去冗餘）。所以官方權重是給你「用」的，這個 repo 是給你「懂」的，兩件事不衝突。

## 進階與可能被釘

⚠️ **Q：你說 BYOL、SimSiam 完全不用負樣本，但我看過有人說它們其實偷偷藏了「隱性負樣本」，是不是根本沒真的拿掉？**
**A：** 這是個超好的問題，而且學界真的吵過。當年有一篇很有名的分析說：BYOL 之所以不坍塌，關鍵其實是 batch normalization——因為 BN 會用整個 batch 的統計量去標準化，等於偷偷把別的樣本當參照，所以「隱性負樣本」其實還在。但後來 BYOL 團隊回應、做了實驗：把 BN 換成不看 batch 的 normalization，它一樣不坍塌。所以目前比較被接受的共識是：真正撐住它的是「打破對稱」——這裡有兩個機制分工，predictor 只加在其中一支上製造不對稱，stop-gradient 則切斷另一支的梯度回傳，兩者合起來讓兩邊不可能塌到同一個常數解，而不是靠什麼隱藏的負樣本。SimSiam 那篇就是把東西砍到只剩 stop-gradient 來證明這件事——它甚至把 BYOL 的 EMA 動量編碼器都拿掉了，照樣不坍塌，說明 EMA 其實不是必要的。所以負樣本是真的拿掉了，只是「防坍塌」這個任務交給了別的機制。

⚠️ **Q：你一直說「防坍塌」是真正的難題、負樣本只是其中一條路。那有沒有一個統一的理論，能解釋為什麼這麼多不同招式都能防坍塌？**
**A：** 目前有不少嘗試，但老實說還沒有一個大家都點頭的「大一統理論」。比較有共識的一個視角是（Wang & Isola 2020）：防坍塌其實是在管兩股力量——一股把同一張圖拉在一起，叫 alignment；一股逼特徵盡量攤開、佔滿整個空間，叫 uniformity。你回頭看就會發現：負樣本是用「推開」直接製造 uniformity；SwAV 用 Sinkhorn-Knopp 做的是「軟分配」、逼這些碼盡量均勻地用到每個 prototype（注意它不是 k-means 硬分群，而且還搭一個 swapped prediction——一個視角去預測另一個視角的碼）；Barlow Twins 用去相關；DINO 用 centering 不讓某一類獨大——它們都是在用不同手段補上那股「攤開」的力。所以與其說有統一理論，不如說有一個統一的「視角」：不管哪招，都要同時做到拉得緊、又攤得開，只是各自選了不同的工具。這也正好呼應我們報告的 punchline。

⚠️ **Q：既然自監督這麼強，那專門的「監督式對比學習」（SupCon 那種）還有存在意義嗎？有標籤的時候要選哪個？**
**A：** 有意義，而且定位很清楚。SupCon（Khosla et al., NeurIPS 2020）的想法是：當你「有標籤」的時候，別浪費——它把「同一類的所有圖」全都當成正樣本一起拉近，而不是像 SimCLR 只認同一張圖的兩個視角。結果就是分類常常比直接用 cross-entropy 還穩、對雜訊標籤更耐操。所以選擇很簡單：標籤多就用 SupCon 這種監督式對比，把標籤的價值榨乾；標籤少或沒有，就走 SimCLR、BYOL 這條自監督路。有趣的是它們是同一個家族——SupCon 在沒有標籤、退化成每張圖只有一個正樣本時，公式就剛好變回 SimCLR。所以與其說是對手，不如說是同一條式子在「有沒有標籤」兩種情境下的兩個面貌。

**Q：對比學習講得這麼厲害，那它跟傳統「有標籤的監督式學習」到底差在哪？是要取代監督式嗎？**
**A：** 其實不是取代，比較像分工。監督式是「老師直接告訴你答案」——這是貓、這是狗，模型照著標籤學；對比學習是「沒老師，自己從資料裡找規律」，靠的就是同一張圖拉近、不同圖推開。兩者最大的差別是：監督式學到的特徵是「為了那個任務量身訂做」，換個任務就有點水土不服；自監督學到的是比較通用的語意，反而換任務時更好用。所以現在主流做法是兩段式——先用海量無標籤資料自監督預訓練，再用少量標籤微調，等於先讓它自己長出常識，老師最後再點一下。這條路其實我們報告裡 SimCLR v2 就示範過了：它證明大型自監督模型其實是很強的半監督學習者，先用無標籤把大模型 pretrain 好、再用極少量標籤微調、最後蒸餾到小模型。

**Q：現在都 CLIP、GPT、大型基礎模型的時代了，這種純影像的自監督對比學習還重要嗎？是不是已經過時了？**
**A：** 完全沒過時，反而是它的精神贏了。你想，CLIP 本質上就是一個跨模態的對比學習——把「圖」和「對應的文字」拉近、不對應的推開，骨子裡還是拉近推開那套。LLM 那邊雖然主力是「預測下一個字」，但精神一樣是「不靠人工標籤、從資料自己生出學習信號」。而且純影像這條線今天也長成了 DINOv2 這種視覺基礎模型，特徵不用再訓練就能接分類、分割、深度估計。所以與其說過時，不如說對比學習是這整波 foundation model 的思想源頭之一，它只是換了舞台繼續演。

**Q：今天講的都是影像。這套拉近推開、防坍塌的東西，搬到文字、語音、多模態也適用嗎？**
**A：** 適用，而且已經被大量用了。文字那邊有 SimCSE，做法幾乎就是 NLP 版的 SimCLR——同一個句子丟兩次、用 dropout 當「增強」，得到兩個略微不同的版本去拉近，別的句子推開，效果好到變成句向量的標準招。語音、影片也都有對應的對比方法。重點是：這套框架本身跟「資料是圖還是字」無關，它只要求你能定義「什麼是同一個東西的兩個視角」跟「什麼是不同的東西」。唯一真正換湯的地方，是「增強」要重新設計——圖片可以裁切翻轉，句子就得用回譯、dropout、換同義詞這種方式。所以可以說：核心哲學跨模態通用，難的是替每個領域找到對的「視角」。

**Q：這些方法都號稱能不靠標籤學出語意，那它有沒有什麼明顯的失敗模式或限制？什麼時候它會學壞？**
**A：** 有，而且很現實。第一個就是我們全場在講的坍塌——一沒設計好，模型就偷懶把所有東西對應到同一個點，SimSiam 那邊只要忘了 stop-gradient，幾乎是開場沒幾步、loss 就一路掉到負餘弦相似度的下界 -1，直接躺平給你看。第二個是它非常吃「增強策略」：增強選錯，模型會學到捷徑，比如靠顏色、靠紋理分類，而不是真的懂語意，這正是 InfoMin 在處理的痛點——它的核心是「最小充分視角」：兩個視角之間既不能共享太多、也不能太少資訊。第三個是資源門檻，像 SimCLR 要超大 batch、DINOv2 要上億張精選資料，一般實驗室玩不動。最後一個比較少人提：它會忠實地把資料裡的偏差也一起學進去——資料偏哪邊，特徵就偏哪邊，因為根本沒有標籤幫它校正。

**Q：講了五年十幾個方法，那接下來呢？這個領域未來會往哪走？還有沒有「拐杖」可以拿掉？**
**A：** 我自己看有幾個方向。第一，從「對比」走向「生成式」自監督——像遮住一塊叫模型補回來的 masked modeling，DINOv2 裡的 iBOT 就是這個味道，它不靠拉近推開也能學。第二，是多模態統一，讓圖、文、語音共用一套表徵，CLIP 只是開頭。第三，是「資料工程」變成主角——DINOv2 證明了與其發明新 loss，不如把資料篩好、放大規模，大力出奇蹟。至於還有沒有拐杖可拿？我會說真正還沒拿掉的，是「人類設計的增強」這根——現在每個領域都還得手工調增強，如果哪天模型能自己學出該怎麼變化視角，那會是下一根被抽掉的拐杖。
